How It Works with Databricks

The stored procedures (`discover_fraud_network` and `get_claim_relationships`) can be called directly by AI agents:

1. Agent receives alert about suspicious claim
2. Agent calls `get_claim_relationships(claim_id)` to understand direct connections
3. Agent calls `discover_fraud_network(claim_id, max_depth=3)` to explore full network
4. Agent analyzes results:
   - Network size and fraud density
   - Total exposure amount
   - Key players (high-degree nodes)
5. Agent makes decision:
   - Low risk: Log and monitor
   - Medium risk: Flag for analyst review
   - High risk: Immediate escalation with full network report
